Library metadata API. Spec is not stable. More info

Use Accept: application/json (or add ?format=json) to get the JSON representation of a resource. More formats might be available in the future. HTML representations are quite limited currently.

Document

A document is some entity open to bibliographic description. Documents include books, journals, media, etc. In addition to bibliographic information, documents embed holdings information describing physical copies of the document.

GET /documents/:id

Returns a single document by its locally unique :id parameter. If not found, a 404 response is returned.

Examples:

  • Get the document with :id=54a9c9e58b70d51c2a0041d9
    /documents/54a9c9e58b70d51c2a0041d9
  • A document demonstrating complex journal succeeding entry:
    /documents/54aaa03c8b70d51c2a0041e5
  • A document demonstrating relator codes for creators:
    /documents/54aaa87f8b70d51c2a0043d4

GET /documents?q=:query

Returns a list of (zero or more) documents, usually filtered by some query. The query is made up of one or more fields.

Examples:
  • id : Query by some identifier, such as
    • BIBSYS objektid:
      /documents?q=id:132038137
    • BIBSYS dokid:
      /documents?q=id:12k189510
    • BIBSYS knyttid:
      /documents?q=id:050076NA0
    • ISBN (ISBN10 or 13, hyphens optional):
      /documents?q=id:1-107-01395-X
  • CQL query: (syntax subject to change)
    • Return the volumes of a multi volume work:
      /documents?q=bs.serieobjektid%3D042457270

Subject

A subject is some entity that is part of a controlled subject headings vocabulary. A subject can be uniquely identified by it's identifier or the combination of an index term and a vocabulary name, see examples below. Subject resources embed document usage information.

GET /subjects/:id

Returns a single subject by its locally unique :id parameter. If not found, a 404 response is returned.

Examples:

  • x:
    /subjects/x

GET /subjects?q=:query

Returns a list of (zero or more) subject headings, usually filtered by some query. The query is made up of one or more fields.

Examples:
  • The term "Steganografi" in the vocabulary "Realfagstermer", as identified by the LC subject source code "noubomn":
    /subjects?q=vocabulary:noubomn+term:Steganografi
